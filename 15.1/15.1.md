# Домашнее задание к занятию «Организация сети»

## Подготовка к выполнению задания

1. Создан аккаунт в Yandex Cloud.

2. На локальный хост установлен Terraform:
   - Скачан архив с [официального сайта Terraform](https://developer.hashicorp.com/terraform/install).
   - Бинарный файл `terraform.exe` помещен в `C:\Windows\System32`.

3. Создан файл `.terraformrc` в домашней директории пользователя со следующим содержимым:

```hcl
provider_installation {
  network_mirror {
    url = "https://terraform-mirror.yandexcloud.net/"
    include = ["registry.terraform.io/*/*"]
  }
  direct {
    exclude = ["registry.terraform.io/*/*"]
  }
}
```

4. Создан сервисный аккаунт (SA) согласно [инструкции](https://yandex.cloud/ru/docs/iam/operations/sa/create).
5. Сервисному аккаунту назначена роль согласно [инструкции](https://yandex.cloud/ru/docs/iam/operations/sa/assign-role-for-sa).
6. Сгенерирован авторизованный ключ и сохранен в файл `authorized_key.json`:

```bash
yc iam key create --service-account-name <имя_сервисного_аккаунта> --output authorized_key.json
```
7. Создать пару ключей для подключения по ssh.

## Развернутая инфраструктура

### Сеть (VPC)

- **VPC Network**: `lab-network`
- **Публичная подсеть** (`public`): `192.168.10.0/24`, зона `ru-central1-a`
- **Приватная подсеть** (`private`): `192.168.20.0/24`, зона `ru-central1-a`

### Виртуальные машины

| Имя | Роль | Внутренний IP | Публичный IP | Образ |
|-----|------|---------------|--------------|-------|
| `vm_nat` | NAT-инстанс (шлюз) | 192.168.10.254 | Да | `fd80mrhj8fl2oe87o4e1` |
| `vm_public` | Тестовая ВМ в публичной сети | Динамический | Да | Ubuntu 24.04 LTS |
| `vm_private` | Тестовая ВМ в приватной сети | Динамический | Нет | Ubuntu 24.04 LTS |

### Маршрутизация

Создана таблица маршрутизации для приватной подсети со статическим маршрутом:
- **Destination**: `0.0.0.0/0` (весь интернет-трафик)
- **Next hop**: `192.168.10.254` (NAT-инстанс)

Это обеспечивает доступ в интернет для виртуальных машин в приватной подсети через NAT-инстанс.


## Проверка работоспособности

### 1. Подключение к публичной ВМ

```bash
ssh -i ~/.ssh/id_ed25519_yd ubuntu@<public_vm_ip>
```

### 2. Подключение к приватной ВМ через публичную (ProxyJump)

```bash
ssh -J ubuntu@<public_vm_ip> ubuntu@<private_vm_ip>
```

Примечание: У меня возникли проблемы при подключении по ssh через jump host, вероятно из-за того, что опция -i применяется только ко второму соединению (целевой хост), но не применяется к первому (прыжковый хост). У меня не получилось подключиться так:
```bash
ssh -J ubuntu@<public_vm_ip> ubuntu@<private_vm_ip> -i ~/.ssh/id_ed25519_yd
```
Поэтому я добавил ключ в агент:

1. Настройка SSH-агента (для Windows). Откройте PowerShell от имени администратора и выполните:

```powershell
Set-Service ssh-agent -StartupType Automatic
Start-Service ssh-agent
```

2. Добавление ключа в агент:

```powershell
ssh-add C:\Users\username\.ssh\id_ed25519_yd
```
3. Подключение к приватной ВМ через публичную (ProxyJump):

```powershell
ssh -J ubuntu@<public_vm_ip> ubuntu@<private_vm_ip>
```

### Задание 1. Yandex Cloud 

**Что нужно сделать**

1. Создать пустую VPC. Выбрать зону.
2. Публичная подсеть.

 - Создать в VPC subnet с названием public, сетью 192.168.10.0/24.
 - Создать в этой подсети NAT-инстанс, присвоив ему адрес 192.168.10.254. В качестве image_id использовать fd80mrhj8fl2oe87o4e1.

 - Создать route table. Добавить статический маршрут, направляющий весь исходящий трафик private сети в NAT-инстанс.
 - Создать в этой приватной подсети виртуалку с внутренним IP, подключиться к ней через виртуалку, созданную ранее, и убедиться, что есть доступ к интернету.
